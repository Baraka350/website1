A webpage is a single document that is part of a website. It can contain text, images, videos, links, and other multimedia elements that are displayed in a web browser. A webpage is accessed using a unique URL (Uniform Resource Locator).

Think of a webpage like a single page of a book. It provides specific information, and you can navigate through other pages of the website by clicking on hyperlinks.

  • Part of a larger website.

  • Can be static or dynamic, depending on the design.

Domain name

A domain name is the unique address of a website on the internet. It is what users type into a web browser to access a website instead of using an IP address.

Top Level Domain (TLD)

A Top-Level Domain (TLD) is the last part of a domain name, appearing after the final dot (.) in a URL. It represents the highest level in the domain name system (DNS) hierarchy and helps classify domain names based on their purpose, geographic location, or type.

How does webpage work?

When a user enters a URL (Uniform Resource Locator) into a web browser, the browser sends a request to a web server to retrieve the webpage associated with that URL. The web server processes the request and sends back the webpage data, which the browser then renders and displays to the user.

Webpages are typically written in HTML (HyperText Markup Language) and may include other technologies like CSS (Cascading Style Sheets) for styling and JavaScript for interactivity. These technologies work together to create the visual and interactive elements that users see and interact with on a webpage.

Webpages can contain various types of content, such as text, images, videos, links, forms, and interactive elements. They are designed to provide information, engage users, and facilitate navigation within a website.

Request SSL: The client (web browser) initiates a connection to the server and requests a secure connection using SSL. This is often indicated by "https://" in the website's address.

SSL Handshake: The client and server perform an "SSL handshake," where they exchange information to establish a secure, encrypted connection. This involves verifying each other's identities and agreeing on encryption methods.

Response: The server sends back the requested website content (HTML, CSS, JavaScript, images, etc.) to the client. This data is encrypted using SSL, ensuring that it remains confidential during transmission.

Web Server & DB Interaction (Optional): If the website requires dynamic content (e.g., a login page, a product catalog), the web server may interact with the database to retrieve or update information.

Display: The client (web browser) decrypts the received data and displays the website to the user.

Types of webpage

There are several types of webpages, each serving a different purpose and catering to specific user needs. Some common types of webpages include:

Static Webpages

A static webpage is a webpage that displays the same content to every user. The content is fixed and doesn't change unless manually updated by the website developer. These pages are typically coded in HTML, and they don't involve any backend programming or databases.

characteristics:

  • Content Consistency: The content remains the same for all visitors and does not change unless the page is edited manually.
  • Speed: Static pages load faster because they are simple and don‚Äôt require server-side processing.
  • Easy to Develop: Static webpages are straightforward to develop using basic HTML and CSS, making them an excellent choice for small websites.
  • No Interaction: These pages don't offer interactive features like user login, form submissions, or dynamic content.
  • Limited Customization: Since they don't rely on databases or backend systems, static pages cannot provide personalized experiences or tailor content to the user.

Dynamic Webpages

A dynamic webpage is a webpage that displays different content to users based on their interactions or preferences. These pages are typically generated on the server-side using programming languages like PHP, Python, or JavaScript. Dynamic webpages can pull data from databases, process user input, and provide personalized content.

characteristics:

  • Content Changes Based on User Interaction: The content on dynamic pages can change based on the time of day, user input, or other variables. For example, a social media feed updates with new posts based on the user's activity.
  • nteractivity: Dynamic webpages support user interaction, such as form submissions, logins, comments, and personalized experiences.
  • Requires Backend Technologies: To display dynamic content, these pages often rely on databases (e.g., MySQL, MongoDB) and server-side technologies.
  • Higher Server Load: Dynamic pages can be slower to load than static pages because they require processing on the server before being displayed to the user.
  • User-Specific Content: They can display customized information depending on the user‚Äôs preferences, actions, or data.
